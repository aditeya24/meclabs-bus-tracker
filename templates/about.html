About MecLabs Bus Tracker

MecLabs Bus Tracker is a project dedicated to improving the public transportation experience in Kerala by providing real-time bus information. We aim to empower commuters with accurate and timely data, reducing the stress and uncertainty often associated with bus travel. By leveraging technology and data, we strive to make public transport a more convenient, reliable, and attractive option for everyone.

The Problem We're Solving

Public bus travel in Kerala is often hampered by a lack of real-time information, leading to several key issues for passengers:

  • Unknown Bus Locations: Not knowing the current location of a bus.
  • Unreliable Arrival Times: Inaccurate schedules making trip planning difficult.
  • Missed Buses: Lack of timely information leading to missed connections.
  • Stressful Waits: Uncertainty about bus arrival times causing anxiety at bus stops.
  • Difficult Trip Planning: Challenges in planning journeys involving multiple buses.

Our Solution: Real-Time Bus Tracking

We're developing a user-friendly website that uses Google Maps APIs to provide real-time bus information. Our platform offers the following key features:

  • Real-Time Bus Tracking on a Map: View the current location of buses on an interactive map.
  • Accurate Estimated Arrival Times (ETAs): Dynamically updated ETAs based on real-time data and traffic conditions.
  • Missed Bus Notifications: Alerts if you've likely missed your bus.
  • "Next Bus" Information: Displaying the ETA of the next bus on your route.
  • Simplified Route Planning: Optimized bus route suggestions based on real-time data.
  • Bus Stop Details: Information about bus stops, including served routes and schedules.

Data Acquisition and Processing

We use two primary methods to gather real-time bus location data:

  • GPS Tracking Devices (High Accuracy): Dedicated GPS units installed on buses for precise location updates.
  • Mobile App (Cost-Effective): A simple app on drivers' or conductors' smartphones to share location data.

Our backend system processes this data to calculate accurate ETAs and provide other real-time information.

Benefits for Commuters

  • Empowered Commuters: Greater control over journeys with accurate information.
  • Improved Time Management: Better planning and reduced waiting time.
  • Increased Public Transport Usage: Making bus travel more convenient and attractive.
  • Data-Driven Optimization: Providing data to improve bus services.

What We're Up To

We are currently in the prototyping phase, building a web-based platform to demonstrate the core functionality of real-time bus tracking. Here's a look at our current progress and the technologies we're using:

Technology Stack

  • Backend: A Flask application with Socket.IO for real-time communication. This handles location data processing and updates.
  • Frontend (Initial): Flask's `render_template` function is used to serve the initial website structure and integrate with the backend.
  • Map Visualization: OpenStreetMap and Leaflet are used to render the live location of buses on an interactive map.
  • Location Data Input: A separate "Send Location" webpage allows for real-time location updates to be sent to the backend. This data is temporarily stored on the server and distributed to the main website via Socket.IO.

Current Functionality

Our current prototype allows users to:

  • View the live location of all available buses on a map.
  • See the route map of a selected bus.
  • Receive live updates of bus locations and associated timestamps.

Planned Enhancements

We are planning to expand the functionality with the following features:

  • Database Integration: Implementing a database to store bus routes, schedules, and other persistent data.
  • Predefined Routes: Defining and storing standard bus routes for improved route planning and display.
  • User Authentication: Adding a login page for the "Send Location" section to verify the identity of the location data sender (e.g., drivers).
  • Estimated Time of Arrival (ETA) Calculation: Implementing algorithms to estimate bus arrival times at various stops.
  • Missed Bus Detection: Developing logic to determine if a user has missed a bus based on real-time data.

Next Steps

Our immediate next steps are focused on:

  • Implementing ETA Calculation: Developing and integrating the algorithms for estimating bus arrival times.
  • Improving Location Accuracy: Refining the data acquisition and processing methods to ensure more precise real-time location updates.
  • Implementing Predetermined Routes: Defining and storing standard bus routes within the system.
